= Keystone的永續性 SLO
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
永續性服務等級目標 (SLO) 旨在將 STaaS 產品的功耗控制在額定功率範圍內，以提供統一儲存（不包括 _Value_ 效能服務等級）和區塊最佳化儲存的效能服務等級實例 (PSLI)，或每個 AFX 叢集的功耗控制在額定功率範圍內。了解 Keystone 的永續性目標如何影響您的 STaaS 設定，以及如果未達到限制，如何獲得服務積分。



== 指標

* *瓦特*：儲存陣列或 AFX 叢集消耗的功率，由監控工具報告。參考最大功耗限制在link:https://hwu.netapp.com["NetApp Hardware Universe"^]中提供。
* * Tebibytes* ：最大值：
+
** 承諾容量 + 性能服務等級的分配突發容量，或
** 假設儲存效率係數為，則為有效部署容量 `2:1`。




若要深入瞭解儲存效率比率、請參閱 https://docs.netapp.com/us-en/active-iq/task_analyze_storage_efficiency.html["分析容量與儲存效率節約效益"^]。



=== 支援的組態

永續性 SLO 適用於具有以下支援組態的標準 Keystone STaaS 訂單：

* 僅限 ONTAP AFF A 系列儲存陣列或 AFX 叢集。
* 僅限雙節點無交換器叢集組態。
* 僅限非混合組態。
* 僅限非 MetroCluster 組態。
* 在所有儲存陣列上全面、持續地啟用監控工具和 AutoSupport （ASUP）。
* 功耗測量期間的環境工作溫度必須在 25°C 至 27°C 之間。

=== 服務信用申請

如需申請服務積分，請在服務月份結束後的 7 天內向 NetApp Keystone 支援部門提交優先順序為 3 的（P3）支援工單，並按如下 Excel 表格所示提供詳細資訊：

image:sla-breach.png["Excel 工作表範本"]

請注意以下服務積分申請條件：

* NetApp 判斷是否發生了服務等級違規事件。
* 您必須按 NetApp 要求提供其他資訊，包括您的功率消耗資料和測量方法。
* 服務點數僅根據已承諾容量計算。
* 服務抵扣額僅可用於抵扣下個月的帳單。已過期的 Keystone 訂閱不適用。如需聯絡 NetApp 支援團隊並提交申請服務抵扣所需的支援案例，請參閱link:../concepts/support-services.html["取得 Keystone 的說明"]。
