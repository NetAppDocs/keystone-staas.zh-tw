= Keystone的性能 SLO
:hardbreaks:
:allow-uri-read: 
:nofooter: 
:icons: font
:linkattrs: 
:imagesdir: ../media/


[role="lead"]
Keystone STaaS 提供基於延遲的效能服務等級目標 (SLO)，目標是統一儲存（不包括 _Value_ 效能服務等級）、區塊最佳化儲存和 AFX 儲存的每個效能服務等級實例 (PSLI) 的延遲達到第 90 個百分位。了解如何檢查效能、適用哪些規則，以及如果 Keystone 未達到 Keystone 訂單條款和條件中規定的已使用容量延遲目標（最高可達突發限制），如何獲得服務補償。



== 指標

* * 效能降級 * ：每個事件的時間量（以分鐘為單位）、在這段時間內無法達到 90 個 ^ 百分位數延遲目標。
* *第 90 百分位延遲* 是針對 Keystone 訂單中所有磁碟區的每個磁碟區、每個效能服務等級執行個體進行測量的。延遲每五分鐘採樣一次，並以 24 小時內計算出的第 90 百分位值作為每日指標，同時考慮以下幾點：
+
** 在收集度量時記錄至少五個 IOPS 的磁碟區會被視為範例。
** 在收集度量時、寫入作業超過 30% 的磁碟區會從範例中排除。
** 對於已啟用 FabricPool 的磁碟區、不會計算資料往返目標（冷）儲存設備的傳輸所產生的延遲。
** ONTAP 叢集以外的應用程式、主機或客戶網路所造成的延遲不計入在內。
** 使用進階資料保護附加服務時，目標延遲僅包含本機儲存陣列的 IO 作業。
** 在 24 小時內，至少應有十個有效指標。如果未滿足此條件，則這些指標將被捨棄。
** 在 FabricPool 組態中、當所有要求的資料區塊都位於 FabricPool 來源（熱）儲存設備上、而且來源儲存設備不在 SnapMirror 同步關係中時、效能等級就會適用。


* 僅適用於 NetApp AFF 和 AFX 平台。
* Keystone 訂單中規定的每個效能服務等級執行個體的所有最小和最大容量及其他限制均適用。
* 所有效能服務等級執行個體都需要完全且持續地啟用監控工具和 AutoSupport （ASUP）。




== 服務點數


NOTE: SLA 和保證均以提名方式提供。

NetApp 會針對效能降低問題、提供服務點數：

|===


| * 效能臨界值 * | * 服務額度 * 


 a| 
90 個 ^ 百分位數延遲 > 目標延遲
| 每個日曆發生日的 3% 
|===


=== 服務額度計算

服務額度是使用下列公式來決定：

服務額度 = （受影響容量 / 總承諾容量） X 容量費用 X 受影響天數 X 信用百分比

其中：

* * 受影響容量 * ：受影響的儲存容量量。
* *總承諾容量*：Keystone 訂單的效能服務等級的承諾容量。
* * 容量費用 * ：根據 Keystone 訂單、受影響效能等級的費用。
* * 受影響的天數 * ：受影響的日曆天數。
* * 信用百分比 * ：服務信用的預定百分比。


* 範例 *

以下範例展示如何計算服務積分：

|===


| * 服務層級 * | * 受影響容量 * | * 總承諾容量 * | * 容量費用 * | *受影響的日曆日* | * 信用百分比 * 


 a| 
極致
| 10 層 | 50 層 | 1、000美元 | 2 | 3% 
|===
使用公式：服務額度 = （ 10 / 50 ） X 1000 x 2 x 0.03

根據計算，服務積分為 $12。



=== 服務信用申請

如果偵測到違反 SLA 的情況，請在 NetApp Keystone 支援下開啟優先順序 3 （ P3 ）支援服務單。

* 以下是必要的詳細資料：
+
.. Keystone 訂閱編號
.. Volume 與儲存控制器詳細資料
.. 問題的網站、時間、日期和說明
.. 延遲偵測的計算時間
.. 測量工具與方法
.. 任何其他適用文件


* 請在 Excel 表格中提供如下所示的詳細資訊，以便向 NetApp Keystone 支援部門開立 P3 工單。
+
